
# 高速な診断ツールとしてのPowerShell

今回は、 Windows マシンが侵害された疑いがある場合に、悪意のあるソフトウェアがインストールされていないかを簡単にチェックできる、便利なスクリプトをいくつかご紹介します。

この記事を書くきっかけとなったのは、数ヶ月前に発生した Notepad++ のハイジャック事件です。攻撃者が、侵害されたプラグインとアップデートの仕組みを悪用して、アプリケーションのディレクトリに不正な DLL を送り込みました。 Notepad++ のような信頼されたアプリケーションが悪意のあるDLLを読み込んでしまうと、標準的なセキュリティアラートを簡単にすり抜けてしまう可能性があります。

幸いなことに、ちょっとした生存確認（サニティチェック）を行うために、重くて動作の遅いセキュリティスイートが常に必要というわけではありません。 PowerShell は Windows に標準で組み込まれており、非常に高速なネイティブの診断ツールとして機能します。今すぐ実行してシステムを調査できる、簡単なスクリプトをいくつか紹介します。

### 特定のファイルの確認

脅威インテリジェンスのレポートやセキュリティブログによって、侵害に関連する特定の悪意のあるファイル名や不正なDLLが特定されている場合、最初のステップはローカルドライブ内を捜索することです。

次のコマンドは、 `C:\`  ドライブ内を再帰的にスキャンして特定のファイル名を探します。アクセス権限のエラーは自動的に無視されるため、ターミナルが赤文字のエラーメッセージで埋め尽くされることはありません。

```
&copy&
Get-ChildItem -Path C:\ -Recurse -Filter malware.dll -ErrorAction SilentlyContinue

```

不審なファイルが見つかり、それが実際に悪意のあるものかどうかを確認する必要がある場合は、そのファイル固有の暗号化フィンガープリント（ハッシュ値）を生成して、 VirusTotal などのデータベースと照合できます。

```
&copy&
Get-FileHash -Path "C:\path\to\file" -Algorithm SHA256

```

### 現在のネットワーク接続の確認

マルウェアは多くの場合、指示を受け取ったりデータを外部に送信（エクスフィルトレーション）したりするために、C2（コマンド＆コントロール）サーバーと通信する必要があります。既知の悪意のあるIPアドレスのリストがある場合は、マシンが現在それらとアクティブに通信しているかどうかを確認できます。

```
&copy&
Get-NetTCPConnection | Where-Object {
    $_.RemoteAddress -in @(
        "127.1.1.0",
        "127.1.1.0",
        "127.1.1.0",
        "127.1.1.0"
    )
}

```

### DNSキャッシュの確認

コマンドを実行したまさにその瞬間にマルウェアがアクティブに通信していなくても、 Windows はコンピューターが最近検索したドメイン名の履歴を一時的に保持しています。 DNS クライアントキャッシュを照会することで、マシンが最近、既知の悪意のあるドメインに接続しようとしたかどうかを確認できます。

```
&copy&
Get-DnsClientCache | Where-Object {
    $_.Entry -match "this|that|something"
}

```

### おわりに

PowerShellは、リアルタイムのインシデント対応において非常に汎用性の高いツールです。これらのような基本的なハンティング用スクリプトをいくつか手元に用意しておくことで、単なる「受動的なユーザー」から、自身のマシンを「能動的に守る防衛者」へと変わることができるのです。
